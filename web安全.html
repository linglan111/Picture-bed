  • SQLMap 是一个自动化的 SQL 注入工具，其主要功能是扫描、发现并利用给定 URL 的 SQL 注入漏洞。

    • 使用-p指定参数，或者使用*标记数据包中待测试的参数
    • 借助sqlmap选项--sql-query结合手工注入语句
    • 查表 sqlmap.py -r C:\Users\WW\Desktop\1.txt -p uname -D "security" --tables
    • 查列（字段） sqlmap.py -r C:\Users\WW\Desktop\1.txt -p uname -D "security" -T "users" --columns
    • 自动搜索表单方式查询：sqlmap.py -u "http://127.0.0.1/Less-11/index.php" --forms
    • -h 显示帮助信息-u 目标URL–data POST请求数据–cookie HTTP Cookie头–level 检测等级–risk 风险等级–threads 线程数–current-user 显示当前用户–current-db 显示当前数据库–dbs 显示所有数据库–tables 显示指定数据库中的所有表–columns 显示指定表中的所有列–dump 导出指定表中的所有数据

  • HTTP（超文本传输协议）是一种用于传输超媒体文档（例如HTML）的应用层协议。。HTTP协议永远是由客户端发起请求，服务端给与响应。

  • HTTP协议的组成包括：请求报文、响应报文、状态码、请求方法、响应头、请求头、实体主体和实体头部等。

  • HTTP协议的作用包括：建立连接、传输数据、断开连接等。

  • XSS全称是Cross Site Scripting即跨站脚本，当目标网站目标用户浏览器渲染HTML文档的过程中，出现了不被预期的脚本指令并执行时，XSS就发生了。

    • XSS有三类：反射型XSS(非持久型)、存储型XSS(持久型)和DOM XSS
    • 发出请求时，XSS代码出现在URL中，作为输入提交到服务器端，服务器端解析后响应，XSS代码随响应内容一起传回给浏览器，最后浏览器解析执行XSS代码。这个过程像一次反射，故叫反射型XSS。
    • 存储型XSS和反射型XSS的差别仅在于，提交的代码会存储在服务器端（数据库，内存，文件系统等），下次请求目标页面时不用再提交XSS代码
    • DOM-based 型 XSS 攻击是一种基于文档对象模型（Document Object Model，DOM）的漏洞，攻击者通过修改页面的 DOM 节点形成的 XSS，称之为 DOM Based XSS。

  • 在盲注中，攻击者根据其返回页面的不同来判断信息(可能是页面内容的不同，也可以是响应时间不同)。也就是说执行插入的语句后，不管是否执行，是否成功，都不会回显详细的错误信息，不像普通注入一样那么容易判断。但是，注入攻击还是发生了，只是错误信息被屏蔽掉了

  • 盲注可分为三类：Booleanbase 普通盲注Timebase 基于时间的盲注Errorbase 基于报错的盲注

  • php一句话木马： asp一句话木马： <%eval request("value")%>

  • 一句话木马是一种常见的 Web 安全攻击方式，它利用文件上传漏洞，往目标网站中上传一句话木马，然后你就可以在本地通过中国菜刀 chopper.exe 即可获取和控制整个网站目录。关于网站控制工具，有很多工具可以用来控制网站，例如冰蝎（Behinder 3.0）、蚁剑（AntSword）、哥斯拉（Godzilla Shell 管理工具）等。eval 函数把接收的数据当作 PHP 代码来执行

  • information_schema数据库是MySQL自带的，它提供了访问数据库元数据的方式（元数据是关于数据的数据，如数据库名或表名，列的数据类型，或访问权限等）。在MySQL中，把 information_schema 看作是一个数据库，确切说是信息数据库。其中保存着关于MySQL服务器所维护的所有其他数据库的信息。如数据库名，数据库的表，表栏的数据类型与访问权 限等。在INFORMATION_SCHEMA中，有数个只读表。它们实际上是视图，而不是基本表。

  • HTTP 状态码是用以表示网页服务器超文本传输协议响应状态的3位数字代码。第一个十进制数字定义了状态码的类型。HTTP 状态码分为五类：信息响应 (100–199)，成功响应 (200–299)，重定向 (300–399)，客户端错误 (400–499)和服务器错误 (500–599)

  • .htaccess 文件是一个纯文本文件，它里面存放着 Apache 服务器配置相关的指令。.htaccess 主要的作用有：URL重写、自定义错误页面、MIME类型配置以及访问权限控制等。主要体现在伪静态的应用、图片防盗链、自定义404错误页面、阻止/允许特定IP/IP段、目录浏览与主页、禁止访问指定文件类型、文件密码保护等

  • 文件截断上传是指攻击者通过修改上传文件的请求，使得服务器只接收到文件的一部分，从而达到绕过文件类型的限制，上传任意类型的文件的目的。其中，00截断是将上传文件名或路径名中使用ascll码值为0的字符（也就是null）来进行截断，%00一般用在URL中用于截断url来进行文件包含，两者原理都一样，都是ascll为0的字符，只是形式不同而已

  • CSRF（Cross-site Request Forgery，跨站请求伪造）是一种针对网站的恶意利用。CSRF攻击可以利用用户已经登陆或已经授权的状态，伪造合法用户发出请求给受信任的网点，从而实现在未授权的情况下执行一些特权操作。攻击者可以通过构造一个后台请求地址，然后引诱受害者去点击或者通过某些途径访问这个地址，从而达到攻击的目的。防御CSRF攻击的方法有很多，比如在表单中添加token、检查Referer等

  • 文件上传漏洞通常由于网页代码中的文件上传路径变量过滤不严造成的，如果文件上传功能实现代码没有严格限制用户上传的文件后缀以及文件类型，攻击者可通过 Web 访问的目录上传任意文件，包括网站后门文件（webshell），进而远程控制网站服务器。

  • PHP序列化是指将一个对象转换成可存储或可传输的格式的过程，反序列化则是将序列化后的数据还原成对象的过程。PHP序列化和反序列化可以用于数据存储、数据传输、缓存等场景.

    • 在PHP语言中常见的系列化和反系列化方式主要有：serialize，unserialize；json_encode， json_decode。
    • 产生原因：1．unserialize函数的参数可控。 2．存在魔法函数。

  • 中间件是指在操作系统和应用程序之间提供服务的软件，常见的中间件有Web服务器、应用服务器、消息队列等。中间件漏洞是指由于中间件的配置不当或者使用不当造成的漏洞。常见的中间件漏洞有：

    • IIS：PUT漏洞、短文件名猜解、远程代码执行、解析漏洞
    • Apache：解析漏洞、目录遍历
    • Nginx：文件解析、目录遍历、CRLF注入、目录穿越
    • Tomcat：远程代码执行、war后门文件部署
